# FableCodex v0.6.0 Release Notes

Status: ready to tag and publish after maintainer approval.

Release date: 2026-07-08

## Install Compatibility

Stable install command after publication:

```bash
codex plugin marketplace add baskduf/FableCodex --ref v0.6.0
```

Verified runtime:

- Codex CLI: `codex-cli 0.128.0`
- Codex Desktop app-server user agent: `Codex Desktop/0.128.0 (Mac OS 15.7.5; arm64)`
- Plugin id: `codex-fable5@fablecodex`
- Skill name: `codex-fable5:codex-fable5`
- Enablement path: add the marketplace source with the CLI, then install and enable the plugin in Codex Desktop. `codex plugin add` is not a public subcommand in the verified runtime.

## Added

- `codex-fable5 doctor` / `healthcheck` for package, wrapper, Python script, local state, stale lock, archived findings, and Codex config diagnostics.
- Live runtime smoke testing with `tools/codex_plugin_smoke.py`.
- Release verification with `tools/verify_release.py`, including `--source-check required` for pinned upstream FABLE-5 source validation.
- Release docs for Codex runtime compatibility, state recovery, security/privacy review, and dogfood evidence.
- Workflow examples for debugging, review-only analysis, migrations, provider bridge setup, long-running ledgers, and release dogfood.

## Changed

- README install guidance now targets v0.6.0 and the verified Codex Desktop enablement flow.
- Goal ledger output now includes clearer status summaries and `codex-fable5 goals summary`.
- Findings status output now explains how each status interacts with the final gate.
- Provider bridge docs now cite current official Codex, LiteLLM, and Anthropic guidance and keep credentials out of project files.

## Fixed

- Removed current install guidance that implied `codex plugin add codex-fable5@fablecodex` was valid.
- Added archive-first state recovery guidance for corrupted ledgers and interrupted forced plan replacement.
- Hardened `codex-fable5 update --ref` against option-like ref values.

## Security

- Added the v0.6 security/privacy review.
- Replaced a real-looking Anthropic-style key placeholder with `<your-anthropic-api-key>`.
- Added tests that scan user-facing files for real-looking key patterns.
- Documented `.codex-fable5/` as local-private state and kept generated `build/` artifacts ignored.
- Verified CI pinned-source fetches do not use an Authorization header.

## Verification Evidence

Release-candidate verification:

```bash
python3 tools/verify_release.py --source-check required
python3 tools/codex_plugin_smoke.py --case local --json
```

Results:

- `python3 tools/verify_release.py --source-check required` passed with `Summary: 8 passed, 0 skipped, 0 failed`.
- `python3 tools/codex_plugin_smoke.py --case local --json` returned `ok: true`, installed/enabled `codex-fable5@fablecodex`, found `codex-fable5:codex-fable5`, reported wrapper `codex-fable5 0.6.0`, and confirmed `debugPromptInputIncludesSkill: true`.

Dogfood evidence:

- `docs/DOGFOOD_REPORT.md` records one multi-step ledger task, one review-only findings gate, and one reproduce-first debugging task.
- Accepted dogfood finding `F001` was resolved before release.

## Residual Limitations

- FableCodex is workflow guidance and helper tooling. It does not clone, unlock, or replace the Fable 5 model.
- Optional provider bridge setup still requires the user's own valid credentials, authorized model access, and gateway configuration.
- Tag creation, push, and GitHub release publication are intentionally deferred until maintainer approval.
